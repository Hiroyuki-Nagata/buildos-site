１日目 - PCの仕組みからアセンブラ入門まで

エディタを使う必要が出てくる、ハッカーを目指す君ならEmacsを使っていると私は信じている。 そして書籍を見るとわかるがいきなり16進数表記の文字列を打ち込む必要が出てくる。なんてこった。 やってみたのだが、Emacsは正直バイナリを編集しにくい。とりあえずテキストで作った16進数をバイナリに変換する 方式でやってみよう。

あと、このサイトでは全ての作業をDebian/sidで行っていく。もちろんだいたいの作業は他のLinuxのディストリビューションでも出来るのではないかと思う。

nhexl-mode

Emacsのデフォルトのバイナリエディタ(hexl-mode)は、ほぼソース表示が目的で編集ができないのでnhexl-modeを ELPA経由でインストールする。これはEmacs-24以降ならば特に手こずることなくインストールできるはずだ。

Emacs上で

M-x package-install nhexl-mode

テキストをバイナリに変換する

出来上がったファイルにUNIXコマンドで変換をかける。

cat helloos.txt | tr -d ' \n' | xxd -r -p > helloos.img
  • catでファイルの中身を標準出力に出す
  • trで空白と改行を削除
  • xxdで16進数をバイナリに変換し標準出力に出す
  • 最後にそれをファイルにリダイレクト

というようなことをやっている。まあ、そんなことをしなくてもGHexのようなバイナリエディタを使えば良さそうなものであるが…

後は出来上がったファイルをEmacsやGHexなどのバイナリエディタで開いて合ってるかどうか確かめるとよさそうだ。

QEMUでブートさせてみる

このバイナリはそのまま起動できない。ファイルの形式はFAT12。 そもそもLinuxで言うところのGRUBがやる部分をわたしたちは書いたのだ。

ちなみにQemuは「ｷｭﾑ」と発音するのが正しそうだ。

このバイナリでブートするための仮想環境を用意する

# apt-get install qemu qemu-kvm

入ったらそれを以下のコマンドで起動する

$ qemu-system-i386 -fda helloos.img

起動しました
